# Системные требования для консольного взаимодействия пользователя через систему меню и диалогов ввода данных. 

# 1. Цель модуля 
Модуль предоставляет универсальные, тестируемые механизмы для организации консольного пользовательского интерфейса:
```
1) Иерархическое текстовое меню (поддержка подменю, возврат на уровень выше, выход из программы).

2) Диалоги ввода данных с валидацией (числа, строки, даты, перечисления, пользовательские типы).

3) Абстракция ввода-вывода для возможности unit-тестирования без участия реальной консоли.
```

# 2. Архитектура проекта
```
1) Input - абстрактный класс, который получает результат в виде строки, в которой записаны данные с ввода данных,

2) ConsoleInput - абстрактный класс, связанный с Input, который записывает данные из консоли в строку в виде текста
через std::cin, данный результат хранится в переменной типа std::string.

3) MenuItem - класс-компонент в меню, который выбирает пользователь.

4) Menu - класс-меню, в котором лежат сами компоненты из MenuItem.

5) MenuRunner - класс, который запускает сам класс Menu для взаимодействи с компонентами.
```
# 3. Принципы проектирования
```
1) Класс Menu не должен работать напрямую с std::cin.
В конструктор передаются ссылки/указатели на Input.

2) Вся логика ввода - через объекты, реализующие интерфейсы.

3) Ресурсы (потоки) управляются автоматически.

4) Каждый публичный метод должен быть изолированно тестируем через mock-объекты.
```
# 4. Требования к тестируемости 
```
1) Mock-объект для Input

2) Покрытие тестами публичных методов

3) Использование Boost.Test

4) Отсутствие прямых обращений к std::cin в тестируемых классах
```
