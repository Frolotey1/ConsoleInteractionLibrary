# Системные требования для консольного взаимодействия пользователя через систему меню и диалогов ввода данных. 

# 1. Цель проекта 
Проект предоставляет тестируемые механизмы для организации консольного пользовательского интерфейса:
```
1) Спец-классы для основного меню взаимодействия с пользователем

2) Диалоги ввода данных с валидацией (числа, строки).

3) Абстракция ввода-вывода для возможности unit-тестирования без участия реальной консоли.
```

# 2. Архитектура проекта
```
1) InputForRegistration - абстрактный класс, имеющий методы run и write_to_database необходимые для класса Registration.

2) InputForAuthorization - абстрактный класс, включающий методы run и get_from_database для класса Authorization.

3) InputForChangeUser - абстрактный класс, содержащий методы run и get_from_database_and_change для класса ChangeUser.

4) InputForDeleteUser - абстрактный класс, представляющий методы run и delete_user для класса DeleteUser

5) Registration - класс, наследующийся от InputForRegistration, переопределяет методы из абстрактного класса и включающий свой метод - check_data. Класс предназначен для регистрации логина и пароля нового пользователя в базе данных "Database.txt".

6) Authorization - класс, наследующийся от InputForAuthorization, переопределяет методы из абстрактного класса. Класс предназначен для авторизации под существующим логином и паролем, проверяя данные из "Database.txt".

7) ChangeUser - класс, наследующийся от InputForChangeUser, переопределяет методы из абстракного класса. Предназначен изменения старых данных пользователя (name,password) на (new_name,new_password), работая с базой данных "Database.txt".

8) DeleteUser - класс, наследующийся от InputForDeleteUser, переопределяет методы из абстрактного класса и предназначен для удаления данных из "Database.txt".

9) DeletePath - класс, который удаляет саму базу данных "Database.txt" во время выхода из программы.

10) LoginSystem - основной класс, с которого начинается программа и включающий методы, отвечающие за запуск реализации того или иного класс для авторизации, регистрации, смены пользователя, удаления пользователя.  

11) MenuItem - класс-компонент в меню, который выбирает пользователь.

12) Menu - класс-меню, в котором лежат сами компоненты из MenuItem.

13) MenuRunner - класс, который запускает сам класс Menu для взаимодействи с компонентами.
```
# 3. Принципы проектирования
```
1) Вся логика ввода - через объекты, реализующие абстрактные классы.

2) Ресурсы (потоки) управляются автоматически.

4) Каждый публичный метод должен быть изолированно тестируем через mock-объекты.
```
# 4. Требования к тестируемости 
```
1) Покрытие тестами публичных методов

2) Использование Boost.Test

3) Отсутствие прямых обращений к std::cin в тестируемых классах
```
